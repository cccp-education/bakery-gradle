== EPIC BKY-LENS : Pattern LENTILLE — Réalité Augmentée pour Bakery

**Statut** : En cours — *P0, cadré session 053*
**Priorité** : P0 — remplace BKY-BKG par un pipeline hybride RAG+KG+ségrégation+règles
**Charge** : 21 pts (6 sous-EPICs) — 12 pts terminés (LENS-1 ✅ LENS-2 ✅ LENS-3 ✅)
**Dépend de** : graphify-plugin (N0, déjà en `implementation`), codebase-contracts (N0)
**Rapport détaillé** : `.agents/sessions/053-audit-lentille-contexte-augmente.adoc`

_Problème_ : Bakery a réinventé la roue du Knowledge Graph. Le scoring maison
(`RelatedArticlesService` : tags littéraux + TF-IDF + co-occurrences bornées) est
strictement inférieur au pipeline augmenté codebase+graphify (RAG pgvector 384-dim +
graphe structurel 203k nœuds + similarité sémantique + règles éditoriales).

_Solution_ : Pattern LENTILLE — 3 couches (ségrégation → enrichissement → budget)
qui combine le graphe structurel (graphify), la similarité sémantique (pgvector),
et les règles métier éditoriales (bakery DSL). Le ratio signal/bruit passe de
~1:30000 (203k nœuds brut) à ~1:1 (4-6 articles pertinents par page).

=== Architecture LENTILLE

----
┌──────────┐    SÉGRÉGATION     ┌──────────────┐   ENRICHISSEMENT   ┌──────────────┐
│ graph.json│───filtre communautés──▶│ site-subgraph│───+RAG+rules────▶│  site-graph   │
│ 203k nœuds│    +type+edges       │  ~200 nœuds │                  │  ~200 nœuds   │
│ 188k edges│                     │  ~300 edges  │                  │  ~500 edges   │
└──────────┘                     └──────────────┘                  └──────────────┘
                                       │                                  │
┌──────────┐                           │                                  │
│ codebase  │───RAG similarité──────────┤                          BUDGET  │
│ pgvector  │   (top-K=20, sim>0.7)     │                     ┌──────────────┐
└──────────┘                           │                     │ maxArticles  │
                                       │                     │ = 4-6/page    │
┌──────────┐                           │                     │ minSimilarity│
│ site.yml  │───règles éditoriales──────┤                     │ = 0.7         │
│ bakery DSL│   excludeDrafts          │                     └──────────────┘
└──────────┘                           │                           │
                                       ▼                           ▼
                                ┌──────────────┐           ┌──────────────┐

=== DSL Proposée

```kotlin
bakery {
    augmentedContext {
        enabled = true
        lens {
            scope = SUBGRAPH                    // SUBGRAPH | FULL | SEMANTIC_ONLY
            communities = ["bakery-gradle", "codebase-gradle", "slider-gradle"]
            nodeTypes = ["file"]
            edgeTypes = ["reference", "agent_reference"]
            maxDepth = 2
            fileExtensions = ["adoc", "md", "html"]
            rag {
                enabled = true
                similarityThreshold = 0.7
                topK = 20
            }
            rules {
                excludeDrafts = true
                excludeTags = ["wip", "draft"]
                prioritizeCrossReferences = true
                crossRefBonus = 0.2
                communityAffinity = 0.3
            }
            budget {
                maxArticlesPerPage = 4
                maxTokensPerArticle = 200
            }
        }
    }
}
```

=== Scoring Hybride

----
score = α × ragSimilarité      // 0.40 — similarité sémantique (pgvector)
      + β × graphProximité     // 0.30 — voisinage structurel (graphify depth)
      + γ × tagRecouvrement   // 0.15 — tags éditoriaux partagés (JBake metadata)
      + δ × crossRefCount     // 0.15 — références croisées AsciiDoc (xref)

  + bonus 0.2 si même communauté (communityAffinity)
  + bonus 0.2 si cross-reference AsciiDoc vérifiée (prioritizeCrossReferences)
----
