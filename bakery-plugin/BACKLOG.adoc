== EPIC 6 : Integration tache publishProfil dans le plugin

**Statut** : A faire
**Priorite** : P1

_Code source de reference_ : `cheroliv.com/build.gradle.kts` (tache `publishProfil` inline + data classes `ProfileSiteConf`/`ProfilePushConf`/`ProfileRepoConf`/`ProfileCreds` + credential resolution `profileCredentials`)

_Objectif_ : Migrer la gestion du profil GitHub du build script inline vers le plugin bakery en tant que tache a part entiere (`PublishProfileTask`), en suivant les patterns existants du plugin (`GitService`, `SiteConfiguration`, `SiteManager`).

=== Analyse du code source a migrer

La tache `publishProfil` dans `cheroliv.com/build.gradle.kts` fait :
. Lit les credentials depuis `site.yml` via Jackson (`ProfileSiteConf` -> `ProfilePushConf` -> `ProfileRepoConf` -> `ProfileCreds`), avec fallback sur les project properties `-PprofileToken` et `-PprofileUsername`
. Copie `README.md` et `README_fr.md` du projet vers un repertoire temporaire `build/cvs-profil/`
. Initialise un depot git dans ce repertoire avec `Git.init()` (branche `main`)
. Ajoute le remote `origin` pointant vers `https://github.com/cheroliv/cheroliv.git`
. Force-push vers `origin/main` avec `UsernamePasswordCredentialsProvider`
. Nettoie le repertoire temporaire
