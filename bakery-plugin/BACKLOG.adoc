==== CS-FIN-1 : Fix bug critique BakeryPlugin — `configPath.get()` sans `isPresent` (CS-21 + CS-16)

*Priorité* : P0 — crash utilisateur si configPath non défini
*Charge* : 2 pts
*Fichiers* : `BakeryPlugin.kt`

_Problème_ : Deux bugs superposés dans l'initialisation du plugin :

. **CS-21** : `bakeryExtension.configPath.get()` (ligne 47) lève `MissingPropertyException` si configPath n'est jamais défini (ni DSL, ni gradle.properties, ni -P). Bug dormant depuis la création du plugin — jamais déclenché car l'usage réel fournit toujours un configPath. Identifié dans `revue_de_code_last.adoc` il y a des mois, jamais corrigé.

. **CS-16** : `isGradlePropertiesEnabled` (ligne 40) est calculé **dans `apply()` avant `afterEvaluate`**. À ce stade, le DSL utilisateur n'est pas encore exécuté → `configPath.isPresent` retourne toujours `false`. Conséquence : `configureConfigPath` cherche dans `gradle.properties` même quand le DSL définit `configPath`, pouvant écraser silencieusement la config DSL.

*Tâches* :

* [x] **CS-21** — Ajouter `isPresent` guard avant `configPath.get()` :
```kotlin
if (!bakeryExtension.configPath.isPresent) {
    logger.warn("configPath not set. Only scaffold tasks available.")
    scaffoldOnly(project, bakeryExtension)
    return@afterEvaluate
}
```
* [x] **CS-16** — Déplacer `isGradlePropertiesEnabled` DANS `afterEvaluate` :
```kotlin
project.afterEvaluate {
    val isGradlePropertiesEnabled = bakeryExtension.configPath.isPresent
    // ...
}
```
* [x] Vérifier que `./gradlew test` passe (BakeryPluginTest + PluginApplicationTest)
* [x] Tester manuellement : appliquer le plugin SANS DSL ni config → pas de crash, message warning
