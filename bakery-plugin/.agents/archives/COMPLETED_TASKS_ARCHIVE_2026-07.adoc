### Leçons

- **`mustRunAfter` vs `dependsOn`** : doux préserve l'indépendance des tâches
  (l'utilisateur peut lancer une tâche seule), dur force la chaîne. Pour un
  pipeline translate→inject, doux est le bon choix — chaque tâche est utile
  isolément.
- **`LauncherDiscoveryRequestBuilder`** : pour itérer sur une feature
  Cucumber isolée sans lancer le cucumberTest complet (5min), utiliser
  `LauncherDiscoveryRequestBuilder.request().selectors(selectClasspathResource(...))`
  + `LauncherFactory.create().execute(request, listener)`. Nécessite
  `junit-platform-launcher` en `testImplementation` (pas `testRuntimeOnly`).
- **Workflow réutilisable `workflow_call`** : factorise 8 workflows déploiement
