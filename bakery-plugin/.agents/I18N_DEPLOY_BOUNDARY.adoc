= I18N_DEPLOY_BOUNDARY — Boundary bakery ↔ sites (EPIC BKY-I18N-DEPLOY)

_Cadré Session 179 (2026-07-18). EPIC BKY-I18N-DEPLOY — US DEPLOY-0._

== Principe

Bakery est une library/plugin Gradle pur. Les sites sont des données
consommatrices. La traduction i18n opère sur des **copies de travail** du
contenu, jamais sur les sources originales.

== Boundary — bakery traduit, les sites déploient

[cols="2,3,3"]
|===
| Couche | Responsabilité | Propriétaire

| Traduction contenu `.adoc`
| Parse → PivotArticle → traduit bloc par bloc → réécrit
| **bakery** (`ContentTranslationService`)

| Classification PlantUML
| TRANSLATE / PRESERVE / BORROW par bloc `[plantuml]`
| **bakery** (`PlantUmlClassifier`, US DEPLOY-1/2)

| Injection RTL
| `:jbake-lang:` + `:lang: rtl` post-traduction (ar/ur)
| **bakery** (`RtlDirectionInjector`, US DEPLOY-3/4)

| Sélecteur de langue
| Fragment Thymeleaf `menu.thyme` + URLs inter-langues
| **bakery** (`injectLangSwitch`, BKY-LANG-SWITCH S177-178, déjà livré)

| Thème / templates métier
| `header.thyme`, `footer.thyme`, CSS
| **site** (pas bakery)

| CI / déploiement
| Bakes séquentiels, deploySite gh-pages
| **site** (workflow GitHub Actions, pattern S031)
|===

== Copies de travail — convention

Pour chaque site, le contenu source FR est dans `content/blog/` (ou
`jbake/content/blog/` selon la structure). Les variantes localisées vont dans
`content-i18n/{lang}/blog/` (créé par `MigrateContentI18nTask`).

----
office/sites/cheroliv.com/
├── doc/refonte/jbake/content/blog/   ← source FR (intact, ne pas modifier)
├── content-i18n/
│   └── en/blog/                       ← copie cassée actuelle (108 fichiers)
│       (ar, bn, es, hi, pt, ru, ur, zh à créer par DEPLOY-7)
└── site.yml
----

*Source FR* : `office/sites/cheroliv.com/doc/refonte/jbake/content/blog/`
(108 articles `.adoc`, format JBake natif = `= Title / @Author / :jbake-*:`).

*Copie EN actuelle* : `office/sites/cheroliv.com/content-i18n/en/blog/`
(108 fichiers, **corrompus** — headers aplatis `title=/date=/~~~~~~` +
contenu FR sur une seule ligne). À écraser proprement en DEPLOY-5.

== Constat — corruption des copies EN actuelles

Les 108 articles `content-i18n/en/blog/` ont été générés par un outil externe
(`cp` + `sed` probablement), pas par `migrateContentI18n`. Conséquences :

. Header JBake natif (`= Title / @Author / :jbake-*:`) écrasé par template
  pivot vide `title=/date=/type=/status=/~~~~~~`.
. Corps FR original aplati sur une seule ligne (newline → espace).
. Aucune traduction — contenu 100% français.

DEPLOY-0 corrige via re-run `migrateContentI18n --contentI18nDryRun=true` (smoke
test parsing, zéro écriture) → DEPLOY-5 apply LLM réel écrase proprement.

== Commande dry-run (DEPLOY-0 smoke test)

----
./gradlew migrateContentI18n \
  --contentI18nSource=doc/refonte/jbake/content/blog \
  --contentI18nOutput=content-i18n \
  --contentI18nSourceLang=fr \
  --contentI18nTargetLangs=en \
  --contentI18nDryRun=true
----

Assertions (smoke FT) :
. `BUILD SUCCESSFUL`
. Logs listent les 108 fichiers parsés
. `content-i18n/en/` n'existe pas (dry-run = zéro écriture)
. Aucun crash parsing

== Commande LLM réel dogfooding (DEPLOY-5 — opérateur, pas CI)

_LLM metered, non-déterministe — ~108 appels pour 108 articles × 1 langue.
À lancer manuellement par l'opérateur, jamais en CI._

----
./gradlew migrateContentI18n \
  --contentI18nSource=doc/refonte/jbake/content/blog \
  --contentI18nOutput=content-i18n \
  --contentI18nSourceLang=fr \
  --contentI18nTargetLangs=en \
  --contentI18nDryRun=false
----

Pré-requis :
. Plugin bakery 0.0.4+ publié mavenLocal (`./gradlew publishToMavenLocal`)
. Ollama actif sur port 11437-11465 (rotation), modèle `gpt-oss:120b-cloud`
. `ia.enabled = true` + `ia.portRange = "11437-11465"` dans la config site
. `content-i18n/en/blog/` existant (copie cassée) sera écrasé proprement

Assertions qualité (post-run, à valider manuellement) :
. 108 articles traduits dans `content-i18n/en/blog/`
. Zéro en-tête cassé (`title=/date=/~~~~~~` disparu)
. `:jbake-title:` présent et traduit (EN)
. Ratio FR résiduel < 5% (mots FR restants dans le corps)
. Blocks `[source]` inchangés (code préservé)
. Blocks `[plantuml]` classés selon DEPLOY-2 (TRANSLATE/PRESERVE/BORROW)
. `injectRtlDirection` (DEPLOY-4) post-traduction pour ar/ur (si ciblé)

Loi de l'Économie d'Encre : re-exécuter la commande sur un `content-i18n/en/`
déjà traduit NE re-traduit PAS (load-before-write, delta vide). Le résultat
est persistant et source-controlled (hors `build/`).

== Injection RTL post-traduction (DEPLOY-4)

Après `migrateContentI18n` (non-dry-run), lancer `injectRtlDirection` pour
injecter `:jbake-lang:` + `:lang: rtl` dans les variantes ar/ur :

----
./gradlew injectRtlDirection \
  --contentI18nOutput=content-i18n \
  --contentI18nTargetLangs=ar,ur
----

Idempotent : re-run = même résultat. LTR (en, fr, zh, ...) → drop `:lang: rtl`.
