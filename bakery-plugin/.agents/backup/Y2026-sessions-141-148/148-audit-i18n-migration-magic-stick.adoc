=== Problème de traduction IA avec gemma4:31b-cloud

Le modèle `gemma4:31b-cloud` (ports 11437-11465) ne produit pas de traductions courtes exploitables. Il retourne des tutoriels d'explication pour chaque phrase.

Exemple de `messages_en.properties` généré :

```properties
menu.1=Depending on the context (most commonly on a website), **Accueil** translates to: Home / Welcome / Reception...
```

Ces valeurs contiennent des sauts de ligne, du markdown et des listes. Inutilisables dans des templates Thymeleaf.

=== qwen3.5:397b-cloud inaccessible

- Présent sur le port 11437 mais requiert un abonnement Ollama payant.
- Ports 11438, 11449, 11450 répondent "OK" sans erreur mais n'ont pas le modèle en cache.

== Recommandations

1. Améliorer le prompt dans `LlmServiceTranslationAdapter.buildPrompt()` pour forcer une réponse courte.
2. Ou ajouter un post-traitement dans `LlmTranslationApplier` pour nettoyer les réponses verbeuses.
3. Ou utiliser un modèle plus obéissant (deepseek-v4-pro:cloud sur ports autorisés).
4. Ou générer sans IA et remplir manuellement `messages_en.properties` avec les traductions de la fixture.
