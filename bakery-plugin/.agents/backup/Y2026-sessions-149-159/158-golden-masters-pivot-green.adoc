== Ce qui a été fait

=== 1. Débogage découverte Gradle (RÉSOLU)

*Le test compilait (bytecode présent) mais n'était pas exécuté par*
`--tests "bakery.pivot.PivotGoldenMasterTest"` *(classe seule).*

**Diagnostic** : Bug Gradle/JUnit 5 avec les noms de méthodes backtick Kotlin
(espaces dans les noms Java). Les 3 autres classes pivot
(AsciiDocParserTest, PivotModelTest, PivotYamlRendererTest) étaient découvertes
car au moins une méthode matchait. PivotGoldenMasterTest n'était découvert qu'avec
le nom complet de méthode :
`--tests "bakery.pivot.PivotGoldenMasterTest.golden master quick start adoc to yaml is byte identical"`

**Workaround** : Utiliser les noms de méthodes complets ou le pattern wildcard
`bakery.pivot.*`. Le filtre `excludeTestsMatching("bakery.scenarios.*")` n'est
pas en cause. `contracts.pipeline.scenarios.CucumberTestRunner` (package non
exclu) génère un warning Cucumber mais n'empêche pas la découverte JUnit Jupiter.

=== 2. Baby steps DDD/TDD — Parser + Renderer + Classifier

==== DDD : TextTranslatableClassifier (nouvelle classe de domaine)

Heuristique de classification `translatable: true/false` pour les segments
`PivotInline.Text`. 10 UT RED→GREEN :

* `natural language text is translatable` → true
* `pure numbers and dashes are not translatable` → false (1, 2, 3, -)
* `sizes with units are not translatable` → false (~8 Go, 8 Go, reste (~48 Go))
* `locale codes are not translatable` → false (fr_FR.UTF-8, : fr_FR.UTF-8)
* `product and desktop names are not translatable` → false (firefox, terminator,
  Ollama, ext4, Xubuntu XFCE)
* `comma-separated package lists are not translatable` → false (nmap, iperf3...,
  docker-ce, docker-ce-cli...)
* `technical command parameters in parentheses are not translatable` → false
  ((bs=4M, conv=fsync))
* `technical comma separator is not translatable` → false (, )
* `human language with FR in parentheses IS translatable` → true (: FR (AZERTY))
* `short technical words are not translatable` → false (reste)

Patterns techniques : `\d+\s*Go\b`, `[a-z]{2}_[A-Z]{2}\.UTF-\d+`, `\bext4\b`,
`\bsquashfs\b`, `\bbs=\d+[A-Z]`, `SDKMAN`, `JDK\s*\d+`, `Ollama`, `firefox`,
`terminator`, `wireshark`, `docker`, `Xubuntu`, `Temurin`, package lists
(`[a-z][a-z0-9+.-]*(?:,\s*[a-z][a-z0-9+.-]*){2,}`).

==== TDD : AsciiDocParser — listes numérotées

Support des listes ordonnées numérotées (`1.`, `2.`, `3.`) en plus des marqueurs
`*` et `.` existants. Nouvelles méthodes :
* `isNumberedListMarker(line)` : regex `^\d+\.\s+.+`
* `listMarkerLength(line)` : longueur du préfixe marker (2 pour `* `/`. `, N+1 pour `N. `)
* Refactoring `parseList` : utilise `listMarkerLength` au lieu de `drop(2)`
* `parseParagraph` : arrête sur les marqueurs de liste numérotée

Test RED→GREEN : `parses numbered ordered list with digit prefix` (3 items,
vérifie ordered=true, contenu de chaque item).

18/18 AsciiDocParserTest PASS.

==== TDD : PivotYamlRenderer — corrections indentation + rendu

* **renderSource** : `yamlScalar(language)` — empty→`""`, non-empty→sans quotes
