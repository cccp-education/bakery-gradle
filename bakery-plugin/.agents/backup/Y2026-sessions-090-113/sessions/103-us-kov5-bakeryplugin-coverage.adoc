== Baby Steps DDD/TDD

=== DDD — Analyse des branches non couvertes

Le `BakeryPlugin.kt` (206 lignes) presente 5 branches critiques dans
`afterEvaluate` :

. `configFile` absent → scaffoldOnly + lifecycle warning
. `configFile` present mais YAML invalide → scaffoldOnly + warn
. `configFile` valide mais `srcPath` et `pushMaquette.from` absents → scaffoldOnly
. `configFile` valide avec repertoire source existant → fullPipeline
. `configFile` valide avec maquette existante → fullPipeline

Les tests existants (`AbsentConfigPathTest`, `PluginApplicationTest`) ne
couvraient que la premiere condition (configPath absent) via
`BakeryTestFixture.createAbsentConfigPath()`.

=== TDD — 3 nouveaux tests

Infrastructure : `BakeryTestFixture.createWithProjectDir(projectDir)` permet
de controler l'emplacement du `projectDir` pour ecrire des fichiers temporaires
via `@TempDir`.

*Test 1* : `runs scaffold only when config file does not exist` —
  verification du log `switching to scaffold only`.

*Test 2* : `runs scaffold only when config YAML is invalid` —
  YAML mal forme (`invalid: [broken: yaml: {{{`), verification du log
  `Failed to parse`.

*Test 3* : `runs scaffold only when site and maquette directories do not exist` —
  YAML valide mais `srcPath: missing_site` + `pushMaquette.from: missing_maquette`,
  verification que l'execution ne leve pas d'exception.

Le test 4 (full pipeline) n'a pas ete implemente en JUnit5 — il necessite
un mock Gradle complet incompatible avec le ProjectBuilder. La couverture
du full pipeline est assuree par les tests fonctionnels
`BakeryPluginFunctionalTests` (89.4% instruction sur ce fichier).
