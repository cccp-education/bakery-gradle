== Code Review V5 — 60 Smells Identifiés

Score global : *6.8/10* (en baisse par rapport au 8.7/10 V4 — l'audit exhaustif
révèle des problèmes que les reviews précédentes n'avaient pas vus).

=== 🔴 P0 — 1 smell critique

- *JsonEscape.kt:39-44* — Ordre de remplacement incorrect (backslash avant
  newline). Un JSON déjà échappé `\n` devient `\\n` après step 1, puis step 2
  cherche des newlines littérales (0x0A) qui n'existent plus. Corruption JSON
  garantie sur séquences échappées.

=== 🟠 P1 — 10 smells importants

- *ArticleGenerator.kt:242-254 + ScaffoldGenerator.kt:182-194* — `slugify()`
  dupliqué (2 copies identiques, 12 lignes avec normalisation accents français)
- *ValidateFirebaseConfigTask.kt:163-210* — Parser JSON maison (50 lignes de
  indexOf/substring) alors que Jackson est déjà une dépendance
- *GenerateArticleTask.kt:208* — `entries.first {}` sans `firstOrNull` → crash
  si typo dans le ton CLI
- *SiteTaskRegistrar.kt:201* — `println()` au lieu de `logger.lifecycle()`
- *ValidateFirebaseConfigTask.kt:140-161* — `auth.apiKey` envoyé en clair dans
  le prompt LLM
- *ResolvedConfigs.kt:15-50* — Bloc `when` de 35 branches, fragile
- *FileSystemManager.kt:165-169* — Nouveau `ObjectMapper` à chaque accès
  (pas de lazy/cache)
- *FileSystemManager.kt:187-193* — `Project.read()` avale les erreurs YAML
  silencieusement → retourne `SiteConfiguration()` vide
- *ConfigPromptM.kt:55* — `resolveWithoutPrompt()` throw au lieu de fallback
- *FirebaseContactFormConfig* — Champs non-nullable sans defaults → crash
  Jackson si absents du YAML

=== 🟡 P2 — 37 smells modérés

Top 5 :
- *SUPPORTED_LANGS* dupliqué 3 fois (ConfigResolver, ScaffoldIntention,
  ArticleIntention)
- *updateProperty()* dupliqué (PropertyUpdater.kt + GenerateThemeTask.kt)
- Pattern résolution CLI>DSL>defaults dupliqué 3 fois (GenerateArticleTask,
  GenerateSiteFromIntentionTask, GenerateThemeTask)
- Pattern injection DSL intention dupliqué 3 fois (ContentTaskRegistrar)
- *ConfigResolver.resolveAll()* — 9 blocs `resolveDomain` répétitifs

Autres : code mort (ConfigPrompts.saveConfiguration, SiteManager.registerUtilityTasks),
champs morts (IaConfig.poolPorts/poolModel), magic numbers (BfsProximityCache),
`var` pour type immutable (FirebaseConfigValidator), YAML par concaténation
string (GenerateSiteFromIntentionTask), texte français hardcodé
(GenerateArticleTask), catch muets sans log (AugmentedContextResolver,
SiteContextCollector), `entries.first {}` unsafe (GenerateArticleTask),
`ConfigPromptM` monade over-engineered jamais chaînée avec flatMap,
`createRepoDir` 6 branches conditionnelles, `copyBakedFilesToRepo` chaînage
`.also{}.let{}.apply{}.run{}` illisible.

=== 🟢 P3 — 12 smells légers

Boilerplate DSL 12 méthodes `Action<X>` (idiomatique Gradle, accepté),
`mutableListOf` inutile (OllamaLlmService), `executePushCommand()` retour
nullable inutile, `initAddCommit()` chaînage over-fluent, `SiteType.fromString()`
throw vs `fromStringOrDefault()` philosophies opposées, `inline fun` inutile
(ContentTaskRegistrar), `applyRules()` filtre "draft" redondant,
`buildSiteYml()` concaténation fragile, `ThemeInjector` injecte toujours
les defaults.
