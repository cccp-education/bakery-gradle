:date: 2026-05-30

== Mission

Enrichir `generateSite` pour proposer un choix du type de site JBake au scaffold :
site blog classique (défaut) ou site minimal (basic).

== Travail réalisé

=== 1. Création de `SiteType` enum (SiteScaffolder.kt)

Types supportés :
- `BLOG` (défaut) — site blog avec articles, tags, archives (backward compat)
- `BASIC` — site minimal (index, about, contact)

Méthodes :
- `fromString(value)` — résolution insensible à la casse, throw si inconnu
- `fromStringOrDefault(value)` — fallback vers BLOG si null/blank/inconnu

=== 2. Ajout de `siteType` à `BakeryExtension`

Nouvelle propriété DSL : `bakery { siteType = "basic" }` (blog par défaut).

=== 3. Création des ressources `site-basic/`

Structure complète JBake pour site minimal :
- `templates/` : header, footer, menu, index, page, feed, sitemap
- `content/` : index.adoc, about.adoc, contact.adoc
- `jbake.properties` : config minimal (render.feed=false, render.archive=false, render.tags=false)

=== 4. Modification de `SiteManager.registerGenerateSiteTask`

- Accepte `siteType: SiteType` (défaut BLOG)
- `resourcePathForType()` : BLOG → "site" (ressource existante), BASIC → "site-basic"
- `createAndConfigureSiteYml()` : écrit le bon `bake.srcPath` selon le type
- `copySiteResources()` : copie la ressource spécifique au type

=== 5. Mise à jour de `BakeryPlugin.kt`

- Extraction du `siteType` via `SiteScaffolder.resolveSiteType(bakeryExtension)`
- Passage à `registerGenerateSiteTask(targetDir, siteType)`

=== 6. Nettoyage (feedback utilisateur)

- Suppression du type `FORMATION` (hors scope — formation = training-gradle)
- Suppression des ressources `site-formation/` créées par erreur
- Suppression des tests associés

== Leçons
