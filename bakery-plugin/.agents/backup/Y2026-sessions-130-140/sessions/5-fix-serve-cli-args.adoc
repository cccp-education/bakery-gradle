= Session 7 — Fix arg CLI JBake dans la tâche `serve`
:date: 2026-04-27
:resultat: OK

== Contexte

Après un crash système, l'utilisateur avait fait 4 screenshots comparant le rendu local (`./gradlew serve`) avec le rendu en ligne (`publishSite`). Le rendu local était visuellement différent : boutons rectangulaires, texte illisible, CSS manquant.

== Diagnostic

La tâche `serve` du plugin Bakery passait les arguments CLI JBake avec une syntaxe incorrecte :

[source,kotlin]
----
// BUG — syntaxe paire clé/valeur
args = listOf("-b", src, "-s", dest)
----

JBake CLI attend des arguments positionnels (`src`, `dest`, puis les flags). Le `-s` était interprété comme un flag booléen qui lance le serveur, pas comme un setter de destination. Résultat : JBake servait un répertoire temporaire au lieu de `build/bake/`.

== Correction

[source,kotlin]
----
// FIX — arguments positionnels + flag en dernier
args = listOf(src, dest, "-s")
----

Fichier modifié : `bakery-plugin/src/main/kotlin/bakery/SiteManager.kt`
