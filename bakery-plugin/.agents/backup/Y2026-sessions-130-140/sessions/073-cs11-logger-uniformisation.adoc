=== CS-11 ✅ — Logger Uniformisation (P2)

*Problème* : `.apply(::println).let(logger::info)` pattern dupliqué dans BakeryPlugin.kt, SiteTaskRegistrar.kt, GitService.kt

*Action* :
- `BakeryPlugin.kt:57-59` → `project.logger.lifecycle("config file does not exists or site and maquette directories do not exist.")`
- `SiteTaskRegistrar.kt:71-73` → `logger.lifecycle("create config file.")`
- `SiteTaskRegistrar.kt:80-82` → `logger.lifecycle("write config file.")`
- `GitService.kt:98-100` → suppression du `println(resultString)` en double (conservation `logger.info` SLF4J)

*Tests* : Compilation ✅, tests ciblés PASS (BakeryPluginTest, SiteManagerTest, GitServiceTest, LensPipelineServiceTest, ProfilePublisherTest)

=== CS-9 ⏭️ SKIPPED — Audit faisabilité

*Problème* : `var` dans LensConfig, LensBudget, LensRules, LensRagConfig, AugmentedContextDsl

*Décision* : **SKIP** — Ces classes sont des DSL builders Gradle (pattern `bakery { lens { communities = ... } }`). Le `var` est idiomatique pour les DSLs Gradle — sans `var`, les blocs de configuration `lens { }` ne fonctionneraient pas. La `copy()` est déjà disponible (data class) et testée. La mutation directe via `.copy()` dans les tests changerait la sémantique des tests DSL. Code review priorité "basse" confirmée.

*Impact* : zéro — pas de modification, pas de régression.
