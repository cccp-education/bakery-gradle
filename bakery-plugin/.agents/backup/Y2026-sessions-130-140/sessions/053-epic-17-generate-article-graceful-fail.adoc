== Objectif
Corriger les 2 derniers scénarios Cucumber en échec (EPIC 17 : generateArticle fails gracefully without IA).

== Analyse

Les 2 scénarios échouaient car :
. **Scénario 1** (`generateArticle task is registered`) : Utilisait `Given a new Bakery project with site configured` de ConfigResolverSteps qui appelle `world.createGradleProject()` (sans IA, mais aussi sans `site/` et `maquette/` dirs). Le plugin prenait la branche scaffolding → `registerGenerateArticleTask` jamais appelé → la tâche n'existait pas.

. **Scénario 2** (`generateArticle fails gracefully without IA config`) : Même problème. Le build échouait avec "Task 'generateArticle' not found" au lieu de `IllegalStateException("LlmService")`.

**Bug #2 déjà résolu** : `IaConfig.enabled = false` par défaut (corrigé dans une session antérieure).

== Solution (Baby Steps DDD/TDD/BDD)

. **Étape 1** — Ajout step Cucumber `@Given("a new Bakery project with site configured but without IA")` dans `GenerateArticleSteps.kt` qui appelle `world.createGradleProjectWithSiteConfigured(iaEnabled = false)`.

. **Étape 2** — Mise à jour du feature `9_generate_article.feature` :
   - Scénario 1 : `Given a new Bakery project with site configured but without IA`
   - Scénario 2 : `Given a new Bakery project with site configured but without IA`

. **Étape 3** — Compilation + `publishToMavenLocal` + vérification Cucumber → 70/70 PASS ✅
