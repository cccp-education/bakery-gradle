== Pivot Architecture — Directive Utilisateur

L'utilisateur a rappelé le pattern de surcharge multi-couche :

```
CLI (-PmaVariable) > gradle.properties > DSL bakery { } > site.yml > defaults
```

L'approche correcte pour BKY-BKG :

1. **RelatedArticlesConfig** dans `SiteConfiguration` (modèle DDD)
2. **RelatedArticlesDsl** dans `BakeryExtension` (DSL `bakery { relatedArticles { ... } }`)
3. **injectRelatedArticlesIntoJbakeProperties()** dans `SiteManager` (pattern des 8 injectors existants)
4. **related-articles.thyme** fragment qui lit `config.relatedArticles*` (Thymeleaf côté serveur)
5. **Intégration dans post.thyme** via `th:replace`

Pas de Jsoup, pas de post-bake HTML, pas de placeholder.
La config passe par l'extension → jbake.properties → Thymeleaf.
