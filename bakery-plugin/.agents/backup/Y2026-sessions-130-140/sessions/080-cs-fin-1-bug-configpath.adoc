* `internal` Kotlin + test dans même module = OK, mais attention aux erreurs d'import (test dans `bakery` importe depuis `bakery.lens` au lieu de `bakery` → faute de copier/coller).
* La fonction `escapeJsonForJavaProperties` résout un vrai bug (CS-13) : l'échappement manuel des backslashes causait un double-échappement dans les JSON contenant des path Windows ou des séquences déjà échappées.
