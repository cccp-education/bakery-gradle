== Objectif
Ajouter une validation pré-push dans `GitService.pushPages()` pour éviter des
erreurs silencieuses quand le repo n'est pas configuré ou que le contenu
est absent.

== Baby Steps DDD/TDD

=== Step 1 — DDD : PrePushValidation sealed class
`GitService.kt` enrichi d'une sealed class :
- `PrePushValidation.Valid` — tout OK
- `PrePushValidation.RemoteNotConfigured` — URL remote vide
- `PrePushValidation.ContentAbsent` — dossier dest vide ou inexistant

Fonction `validatePrePush(destPath, git): PrePushValidation` pure, testable.

=== Step 2 — TDD : 4 tests unitaires RED→GREEN
`GitServiceTest.ValidatePrePushTest` :
- `validatePrePush returns Valid when dest has content and remote is configured` ✅
- `validatePrePush returns RemoteNotConfigured when repository URL is blank` ✅
- `validatePrePush returns ContentAbsent when dest directory is empty` ✅
- `validatePrePush returns ContentAbsent when dest directory does not exist` ✅

=== Step 3 — Refactor : Intégration dans pushPages()
`when (validatePrePush(destPath, git)) { ... }` en début de `pushPages()`.
- `RemoteNotConfigured` → `IllegalStateException` avec message clair
- `ContentAbsent` → `IllegalStateException` avec instruction "run the bake task first"
- `Valid` → log info + poursuite normale
