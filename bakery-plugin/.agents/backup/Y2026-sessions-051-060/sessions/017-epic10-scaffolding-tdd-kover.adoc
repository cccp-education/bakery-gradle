= Session 017 — EPIC 10 Scaffolding (TDD complet) + Kover

_Date_ : 2026-05-07
_Type_ : feat + test + quality
_EPIC_ : EPIC 10 (Scaffolding), EPIC 11 (Optimisation tests)

== Objectif

Implémenter le scaffolding de site via TDD : pouvoir créer un dossier
`office/sites/<siteName>/` contenant `site/`, `maquette/`, `site.yml`,
`.gitignore`, `.gitattributes` quand on applique `initSite` avec les propriétés
`sitesBaseDir` et `siteName` dans la DSL bakery.

== Réalisations

=== Phase RED — Tests écrits avant code (4 fichiers de test)

1. Cucumber feature `6_site_scaffolding.feature` : 5 scénarios Gherkin (UJ1 à UJ5)
2. Step definitions `ScaffoldSteps.kt` : glue Cucumber pour les nouveaux steps
3. Tests unitaires `SiteScaffolderTest.kt` : 10 tests en isolation (resolveSiteTargetDir + validateSiteTargetDoesNotExist)
4. Tests fonctionnels `ScaffoldFunctionalTests.kt` : 5 tests Gradle TestKit end-to-end

=== Modification minimum pour compiler les tests

* `BakeryExtension.kt` : ajout `sitesBaseDir` (Property<String>) et `siteName` (Property<String>)

=== Phase GREEN — Implémentation

1. `SiteScaffolder.kt` (créé) :
   * `resolveSiteTargetDir()` — logique de résolution du dossier cible
   * `validateSiteTargetDoesNotExist()` — validation que le dossier n'existe pas encore
   * `Project.resolveAndValidateSiteTarget()` — extension sur Project

2. `SiteManager.kt` (modifié) :
   * `registerInitSiteTask(targetDir: File = projectDir)` — paramètre optionnel
   * Toutes les méthodes privées (`createAndConfigureSiteYml`, `setupGitIgnore`, `setupGitAttributes`, `copySiteResources`, `injectFirebaseConfigIntoJbakeProperties`) acceptent maintenant `targetDir` au lieu d'utiliser `projectDir` en dur

3. `BakeryPlugin.kt` (modifié) :
   * `apply()` → bloc `afterEvaluate` → appel `SiteScaffolder.resolveSiteTargetDir()`
   * Validation `validateSiteTargetDoesNotExist()` si targetDir != projectDir
   * `registerInitSiteTask(targetDir)` avec le dossier résolu

=== Intégration Kover

* `gradle/libs.versions.toml` : ajout `kover = "0.9.1"` + alias plugin
* `build.gradle.kts` : plugin kover, config (main + functionalTest), rapports HTML/XML, `koverThresholdCheck` (seuil 75%)
* Couverture initiale : 57% instruction (sous le seuil, attendu en phase red)
