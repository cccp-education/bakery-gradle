= Session 030 — BKY-JB-3 Google Forms Embed

== Date
2026-06-01

== Type
feat + test (DDD/TDD/BDD baby steps)

== Sujet
BKY-JB-3 : Composant Thymeleaf Google Forms embed — iframe responsive piloté par config YAML `googleForms:`.

== Objectifs
Créer un fragment Thymeleaf `google-forms.thyme` qui embarque un Google Form en iframe responsive, piloté par la config YAML `site.yml` (clé `googleForms:`). Premier EPIC de la Phase B (services JBake).

== Résultats

=== Étape 1 — DDD : Modèle domaine ✅
- `GoogleFormsConfig` data class dans `SiteConfiguration.kt` (formId, width, height, allowMultiple)
- `GoogleFormsDsl` open class pour le DSL Gradle (`bakery { googleForms { } }`)
- Champ `googleForms: GoogleFormsConfig?` dans `SiteConfiguration`
- 2 UT : parsing YAML avec/sans googleForms

=== Étape 2 — TDD : Fragment Thymeleaf ✅
- `google-forms.thyme` avec iframe responsive (CSS aspect-ratio)
- `th:if="${googleFormsFormId != null and googleFormsFormId != ''}"` pour cacher quand pas config
- 2 UT : iframe avec formId présent, fragment vide sans formId

=== Étape 3 — TDD : Injection config → template ✅
- `injectGoogleFormsConfigIntoJbakeProperties()` dans `SiteManager` (même pattern que Firebase)
- Injection `googleFormsFormId`, `googleFormsWidth`, `googleFormsHeight` dans `jbake.properties`
- Intégration `google-forms.thyme` dans `page.thyme` via `th:replace`
- 2 UT : injection avec/sans config

=== Étape 4 — FT : Tests fonctionnels ✅
- 2 FT dans `ScaffoldFunctionalTests` (GoogleFormsEmbedTest nested class)
- Template déployé dans le site + pas d'injection jbake.properties sans config

=== Étape 5 — BDD : Scénarios Cucumber ✅
- Feature `11_google_forms_embed.feature` (3 scénarios : happy-path, no-config, th:if guard)
- `GoogleFormsSteps.kt` avec step réutilisable `the file {string} should not contain {string}`
- Réutilisation steps existants : `ThymeleafComponentSteps`, `MinimalSteps`
