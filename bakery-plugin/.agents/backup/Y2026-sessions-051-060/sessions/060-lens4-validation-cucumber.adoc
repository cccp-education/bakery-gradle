== Diagnostic des 4 échecs

=== Échecs 1-3 : Feature 17 `17_augmented_context_lens.feature` (3 scénarios)

**Cause racine** : Les steps Cucumber `the bakery DSL defines augmentedContext as enabled/disabled`
utilisaient `Regex("(bakery\\s*\\{)")` pour injecter un bloc DSL APRÈS l'ouverture `bakery {`.
Mais `BakeryWorld.createGradleProject()` génère une seule ligne :

----
plugins { id("education.cccp.bakery") }
bakery { configPath = file("site.yml").absolutePath }
----

La regex insère après `bakery {`, ce qui donne :

----
bakery { augmentedContext { enabled = true ... } configPath = file("site.yml").absolutePath }
----

→ Le `configPath` est après le `}` fermant du bloc augmentedContext = syntaxe invalide Kotlin.
→ Erreur : `Expecting an element` ligne 9.

**Fix** : Création de `DslInjectionHelper.insertBeforeClosingBrace()` qui compte les accolades
pour trouver le `}` fermant du bloc `bakery { }` et insère le nouveau bloc juste avant.
Testé avec 6 UT TDD (single-line, multi-line, nested blocks, edge cases).

=== Échec 4 : Feature 18 `18_subgraph_extraction.feature` Scénario 3

**Cause racine** : Le step `When I extract a subgraph with communities "custom-community"`
utilisait `projectDir.resolve("office/graph.json")` en dur comme `graphFilePath`,
ignorant complètement le custom path `"custom/graph.json"` créé par le
step `Given a graph.json file at custom path "custom/graph.json" with 5 nodes`.
Le fichier `office/graph.json` n'existait pas → `loadGraph` retourne un GraphModel vide → 0 nœuds.

**Fix** : Ajout d'une variable d'état `graphFilePath` dans `LensSteps` mise à jour
par les steps `Given` créant le fichier graph.json. Les steps `When` l'utilisent
avec fallback sur `office/graph.json`.

== Leçons apprises

1. **Regex fragile pour injection DSL** : `Regex("(bakery\\s*\\{)")` ne fonctionne
   que si le bloc bakery est sur plusieurs lignes. Il faut compter les accolades
   pour trouver le bon point d'insertion. → `insertBeforeClosingBrace` robuste.

2. **État partagé Cucumber** : Les steps `When` ne doivent jamais utiliser
   des chemin en dur quand les steps `Given` créent un fichier à un chemin variable.
   → Variable d'état `graphFilePath` avec fallback.
