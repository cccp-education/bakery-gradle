= Session 134 — BKY-I18N-MIG US-4 : Traduction automatisée FR→EN

== Contexte

Session 134 — mission BKY-I18N-MIG US-4 : câbler le `TranslationService` transverse de codebase-gradle dans bakery pour activer le mode auto-LLM de migration i18n.

== Actions réalisées

* **US-4-1** : Ajout dépendance `education.cccp:codebase-plugin:0.0.2` dans `bakery-plugin/build.gradle.kts` et catalogue `gradle/libs.versions.toml` (codebase = 0.0.2, codebase-plugin = 0.0.2).
* **US-4-2** : Création de `LlmTranslationApplier` (`bakery/i18n/LlmTranslationApplier.kt`) — service bakery pur qui consomme `TranslationService` et traduit une `Map<String, String>` FR→EN avec fallback FR sur `TranslationResult.Failure`.
* **US-4-3** : Branchement de `I18nMigrationService` sur `LlmTranslationApplier` quand un `TranslationService` est injecté (mode auto-LLM). Le mode manuel existant (valeurs vides pour EN) reste l'override par défaut.
* **US-4-4** : Création de `LlmServiceTranslationAdapter` (`bakery/i18n/LlmServiceTranslationAdapter.kt`) pour adapter le `LlmService` bakery au contrat `TranslationService` transverse.
* **Tests unitaires** : `LlmTranslationApplierTest` (4 tests) + mises à jour `I18nMigrationServiceTest` (3 nouveaux scénarios auto-LLM).
* **Tests d'intégration** : `AutoTranslationI18nMigrationIntegrationTest` (2 scénarios : traduction auto + fallback sur erreur).
* **Cucumber BDD** : Feature 35 `35_i18n_migration_auto_translation.feature` (2 scénarios) + step `messages_en.properties should have empty values` dans `I18nMigrationSteps.kt`.
* **Build** : alignement du toolchain Kotlin sur Java 24 (`kotlin.jvmToolchain(24)`) pour compatibilité avec `codebase-plugin:0.0.2`.

== Décisions

* Renommage du service `TranslationApplier` → `LlmTranslationApplier` pour éviter la collision avec `I18nTranslationApplier` existant (application manuelle de translations_en.properties).
* Le mode auto-LLM n'est pas activé dans les scénarios Cucumber TestKit car le LLM réel n'est pas disponible ; le comportement est validé unitairement/intégration avec `FakeTranslationService`. La feature 35 teste le mode "pas de traducteur injecté" → messages EN vides.
* Alignement Java toolchain 24 nécessaire car `codebase-plugin:0.0.2` cible Java 24.
