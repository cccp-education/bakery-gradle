== Décisions techniques

. *Séparation readKey/writeKey dans PropSpec* — `LanguageInjector` lit
  `resolver("language", "fr")` mais écrit `updateProperty(lines, "site.language", ...)`.
  Sans séparation, le registry data-driven aurait lu `site.language` (inexistant
  dans `ResolvedConfigs.propertyMap`). `PropSpec(readKey, default, writeKey = readKey)`
  résout élégamment — writeKey default = readKey pour les 8 autres injecteurs.

. *GateOnBoolean avec lambda* — Comments et Newsletter ont des gates différentes
  (`commentsEnabled != "false" || collection != "comments"` vs
  `newsletterEnabled != "false" || provider.isNotBlank()`). Lambda
  `booleanGate: (resolved, resolver) -> Boolean` capture la logique sans
  duplication de code.

. *ConfigInjector fun interface préservé* — Adaptateur vers `InjectorSpec`.
  Permet de garder l'API publique `configInjectors` inchangée pour
  `GenerateSiteService` et `ResolvedConfigsTest`.

. *Pas de BDD Cucumber* — Le refactor est purement interne (API préservée).
  Les tests Cucumber existants (Firebase Auth, Comments, Analytics, Newsletter,
  Theme, Layout, Language) valident le comportement de bout en bout. Pas
  besoin de nouveaux scénarios — le safety net est suffisant.
