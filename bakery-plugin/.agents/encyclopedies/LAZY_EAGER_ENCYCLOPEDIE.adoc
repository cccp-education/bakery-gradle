== Vue d'ensemble

=== Problème Résolu

**Contexte** : Les LLM ont une fenêtre de contexte limitée (~100k-200k tokens).

**Symptômes** :

* Saturation du contexte (trop de fichiers chargés)
* Dilution de l'attention (LLM perdu dans les détails)
* Tokens gaspillés sur des infos non critiques
* Hallucinations (LLM invente car contexte saturé)
* Rappels chronophages (réexpliquer à chaque session)

**Solution** : Séparer l'information en 2 catégories (**LAZY** / **EAGER**)

=== Principes Fondamentaux

[cols="1,4", options="header"]
|===
|Principe |Description

|**LAZY/EAGER**
|Séparer l'information critique (toujours chargée) des détails (chargés sur demande).
**EAGER** = Tableau de bord de voiture. **LAZY** = Manuel du propriétaire.

|**1 session = 1 objectif**
|Une session traite UNE seule tâche (1 fichier, 1 feature, 1 bug).
Durée cible : 15-30 minutes.

|**Archive systématique**
|Chaque session terminée est archivée avec résumé et prochaines étapes.
Rien ne se perd, tout se retrouve.

|**Git manuel**
|L'agent NE JAMAIS committer sans permission explicite.
C'est l'utilisateur qui gère Git.

|**Tests sur demande**
|L'agent NE JAMAIS lancer de tests en fin de session sans demande.
Les tests prennent 1-3 minutes.
|===

=== Analogie

[cols="1,2,2", options="header"]
|===
|Type |EAGER (toujours chargé) |LAZY (sur demande)

|**Voiture**
|Tableau de bord (vitesse, carburant, alertes)
|Manuel du propriétaire (détails techniques, historique)

|**Projet**
|`PROMPT_REPRISE.adoc` + `*_ESSENTIALS.adoc`
|Archives sessions, références détaillées

|**Taille**
|<= 100 lignes total (~5k tokens)
|Illimité (lu uniquement si besoin)

|**Chargement**
|Automatique (début session)
|Sur demande explicite
|===

== Règles Absolues

=== Commits/Git — INTERDICTION FORMELLE

**L'agent NE DOIT JAMAIS** exécuter de commit, push, merge, ou toute commande git
modifiant l'historique **SANS permission explicite de l'utilisateur**.

[cols="1,4", options="header"]
|===
|Autorisé |`git status`, `git diff`, `git log`, `git show` (lecture seule)

|Interdit
|`git add`, `git commit`, `git push`, `git merge`, `git rebase` (sauf ordre explicite)
|===

=== Tests en Fin de Session — INTERDICTION FORMELLE

**L'agent NE DOIT JAMAIS** lancer de tests (`./gradlew test`, etc.) lors de la
procédure de fin de session **SANS permission explicite de l'utilisateur**.

=== Secrets et Credentials — SÉCURITÉ MAXIMALE

**INTERDICTION ABSOLUE** :

* Jamais de secrets en dur dans le code
* Jamais de credentials dans les fichiers de config
* Jamais de tokens API commités dans Git

== Cycle de Vie d'une Session

=== Démarrage de Session

Chargement automatique (EAGER) :
1. `PROMPT_REPRISE.adoc` -> Mission, priorités, contexte
2. `.agents/INDEX.adoc` (section règles) -> Règles absolues
3. `LAZY_EAGER_ESSENTIALS.adoc` -> Principes LAZY/EAGER
4. `*_ESSENTIALS.adoc` (si EPIC active) -> Contexte métier critique

=== Pendant la Session

* **1 fichier à la fois** : Modifier un seul fichier avant de passer au suivant
* **Validation immédiate** : Compiler après chaque modification
* **Contexte léger** : Maximum 50k tokens EAGER (~3000 lignes)
* **Progression visible** : Mettre à jour `PROMPT_REPRISE.adoc` en temps réel
* **`publishToMavenLocal`** : Obligatoire après toute modification du code source

=== Fin de Session — Procédure en 6 Étapes OBLIGATOIRES

**Mots-clés déclencheurs** : "nouvelle session", "je quitte", "session terminée", "a plus tard", "on arrête là"

1. **Créer l'archive LAZY** : `.agents/sessions/{N}-{type}-{sujet}.adoc`
2. **Mettre à jour PROMPT_REPRISE.adoc** (max 70 lignes)
3. **Mettre à jour SESSIONS_HISTORY.adoc** : Ajouter la session terminée
4. **Mettre à jour INDEX.adoc** : Date, session en cours N+1
5. **Mettre à jour TEST_COVERAGE_ANALYSIS.adoc** (si tests modifiés)
6. **Mettre à jour COMPLETED_TASKS_ARCHIVE.adoc**

== Métriques et Seuils

=== Métriques de Session Idéale

[cols="1,3", options="header"]
|===
|Métrique |Cible

|**Durée**
|15-30 minutes

|**Fichiers modifiés**
|1-3 maximum

|**Échanges LLM**
|5-10 messages

|**Contexte tokens EAGER**
|< 50k (~3000 lignes)

|**Fichiers EAGER chargés**
|2-3 maximum
|===

=== Signes Qu'il Faut Changer de Session

* Le LLM répète des erreurs déjà corrigées
* Plus de 3 fichiers modifiés en parallèle
* Conversation > 50 messages
* Tu te dis "il faudrait qu'on reparte sur de bonnes bases"

**Règle d'or** : Mieux vaut 5 sessions de 20 minutes qu'une session de 2 heures.
