== 💡 Boîte à idées pour les Futurs Sprints

Ce document centralise les épopées et les User Stories qui ne sont pas prioritaires pour le sprint en cours mais qui sont envisagées pour les développements futurs. L’objectif est d’alléger le fichier `GEMINI.md` principal pour économiser le contexte tout en gardant une trace claire du backlog à long terme.

=== *🌟 Épopées du Projet*

[arabic]
. *Épopée 10 : Intégration du Formulaire de Contact avec Supabase*
* *Objectif :* Remplacer le formulaire de contact statique par une solution dynamique et fiable qui stocke les soumissions dans une base de données Supabase et envoie des notifications par e-mail en temps réel.
* *Justification Technique :* Le choix de Supabase est motivé par son modèle économique avantageux, son adossement à PostgreSQL — une base de données relationnelle devenue un standard de l’industrie et avec laquelle l’utilisateur est familier — et sa simplicité d’intégration.
* *Modus Operandi :*
[arabic]
.. *Backend Supabase :* Configurer la table `contacts`, activer la sécurité RLS et créer une politique d’insertion publique.
.. *Frontend Client JS :* Intégrer le client Supabase JS dans le site pour permettre la communication avec la base de données.
.. *Logique de Soumission :* Implémenter le script côté client pour capturer les données du formulaire et les envoyer à Supabase.
.. *Retour Utilisateur :* Afficher des messages de succès ou d’erreur clairs à l’utilisateur après la soumission.
.. *Notifications :* Déployer un webhook Google Apps Script qui, une fois appelé, envoie un e-mail de notification formaté via Gmail.
.. *Automatisation :* Créer une fonction et un trigger dans Supabase pour appeler automatiquement le webhook à chaque nouvelle soumission.
.. *Améliorations de Sécurité et de Maintenabilité :* Ajouter des validations avancées (honeypot, regex) et améliorer les notifications (formatage HTML, logging), afin de renforcer la sécurité et la maintenabilité de la solution.
. *Épopée 11 : Couverture de Test pour le Build Gradle (Infrastructure)*
* *Objectif :* Mettre en place une couverture de test robuste pour le script de build Gradle, afin de garantir sa fiabilité et sa maintenabilité.
. *Épopée 12 : Migration du Build Script vers un Plugin Gradle (Infrastructure)*
* *Objectif :* Migrer la logique complexe du script de build principal vers un plugin Gradle réutilisable, en s’inspirant du projet `sample_gradle_plugin-kotlin-dsl` et en adoptant une approche TDD.
* *Contrainte :* L’Épopée 11 (Couverture de Test pour le Build Gradle) doit être entièrement terminée avant de commencer cette épopée.
. *Épopée 13 : Mise en Place de l’Infrastructure RAG (Retrieval-Augmented Generation)*
* *Objectif :* Construire le ``second cerveau'' de l’assistant `e3po` en développant un système RAG basé sur la stack `Kotlin + Gradle + LangChain4j + Supabase`. Ce système indexera le contenu du projet pour fournir un contexte pertinent aux requêtes du LLM.
* *Vision :* Le RAG servira à la fois d’outil d’aide au développement en local et de moteur de contenu pour la génération d’articles de blog via des actions GitHub.
* *Technos Clés :* Kotlin, Gradle, LangChain4j, Supabase (PostgreSQL avec `pgvector`), API Gemini Embeddings.
. *Épopée 14 : Génération de Contenu de Blog Assistée par RAG et CI/CD*
* *Objectif :* Mettre en place une chaîne de production de contenu semi-automatisée qui utilise le système RAG pour générer des brouillons d’articles de blog pertinents et contextuels, et les intègre dans le flux de publication existant via GitHub Actions.
* *Contrainte :* L’Épopée 13 (Infrastructure RAG) doit être terminée.
* *Modus Operandi :*
[arabic]
.. Déclencher un workflow GitHub (`articleGenerator`) manuellement avec un sujet.
.. L’action utilise `gemini-cli-action` pour appeler une tâche Gradle.
.. La tâche Gradle interroge le service RAG pour obtenir un contexte riche.
.. Le prompt enrichi est envoyé à Gemini pour générer un brouillon d’article `.adoc`.
.. L’action commite automatiquement le brouillon dans le dossier `/draft`.
.. Le commit déclenche le workflow de publication existant.
