== DEPLOY-1 — Livré

=== Domaine pur DDD

Fichier `src/main/kotlin/bakery/i18n/plantuml/PlantUmlClassifier.kt` créé :

. `sealed interface PlantUmlStrategy` — 3 variantes :
   `TranslateLabels`, `PreserveTechnical`, `BorrowVocabulary` (data objects)
. `data class PlantUmlBlock(raw: String)` — extrait les labels entre
   guillemets (`labelRegex`), skipe les identifiants techniques dotted
   (`com.example.UserService`), détecte le vocabulaire métier emprunté
   (REAC, AFNOR, DC, TS, RNCP, CP, ECF).
. `class PlantUmlClassifier` — `classify(block): PlantUmlStrategy` :
   BorrowVocabulary si vocabulaire métier → TranslateLabels si labels
   traduisibles → PreserveTechnical sinon. Domaine pur, no I/O, no LLM.

Inspiré de HTOWN EPIC PLT-BOUNDARY (S163-166) — `TranslationStrategy.TRANSLATE/
BORROW/PRESERVE` + `TextClassifier` + `TranslatableText` — adapté au contexte
bakery (labels entre guillemets dans blocs `[plantuml]` d'articles AsciiDoc).
