=== 1. Fix découverte Gradle — DÉBOGAGE CRITIQUE

*Bug* : `PivotGoldenMasterTest` compilait (bytecode présent) mais n'était
jamais exécuté par `--tests "bakery.pivot.*"`. Le task `:test` restait bloqué
indéfiniment.

*Diagnostic* : `contracts.pipeline.scenarios.CucumberTestRunner` (package
non exclu par `filter { excludeTestsMatching("bakery.scenarios.*") }`) est
annoté `@Suite @IncludeEngines("cucumber")`. Quand l'engine cucumber est
exclu via `useJUnitPlatform { excludeEngines("cucumber") }`, ce runner reste
bloqué indéfiniment à la découverte, empêchant `:test` de terminer.

*Fix* : `build.gradle.kts` ligne 101-104 — ajout de
`excludeTestsMatching("contracts.pipeline.scenarios.*")` au filtre `:test`.

*Résultat* : `:test` complet en 1m 1s SUCCESS (vs timeout 5min avant). Tous
les tests unitaires passent, y compris `PivotGoldenMasterTest`.

=== 2. Fix bugs parser — 4 nouveaux UT (RED→GREEN)

==== Bug boucle infinie `parseParagraph`

*Symptôme* : Une ligne `----` flottante (hors bloc source) faisait boucler
`parseParagraph` indéfiniment. Le while s'arrêtait sur `startsWith("---")`
mais `sb.isEmpty()` → `return null to i` (sans avancer `i`) → boucle.

*Fix* : `parseParagraph` retourne `null to i+1` (avance d'un cran) quand
rien n'est consommé, pour skip la ligne non reconnue.

==== Bug `parseSourceBlock` IndexOutOfBounds

*Symptôme* : Si le `----` ouvrant n'était pas trouvé, `contentStart`
dépassait `lines.size` → `subList` crash.

*Fix* : Guard `if (i >= lines.size) return Source(language, "") to i`.

==== Tolérance `====` comme fermeture de bloc source

*Contexte* : Le golden master `scripts.adoc` ligne 79 contient `====` au
lieu de `----` pour fermer un bloc source. Le YAML golden master attend
ce bloc comme `Source` valide.

*Fix* : Nouvelle méthode `isSourceClosingDelimiter(line)` qui accepte
`----` OU `====` comme fermeture de bloc source.

*Tests (4 UT)* :
- `parses level 4 heading with four equal signs and text` (heading `====`)
- `parses source block closed by four equal signs instead of dashes`
- `skips stray four-dashes line without infinite loop`
- `parses source block without closing delimiter reaches end of input`

=== 3. Baby steps TDD — 3 edge cases parser

==== Tables multi-lignes

*Fix* : `parseTableRows` concatène les lignes de continuation (sans `|`)
à la dernière cellule au lieu de les traiter comme nouvelles cellules.

*Test* : `parses table cell spanning multiple lines as single cell` —
vérifie que "Nettoyer le repertoire\nde build avant de construire" est
une seule cellule avec texte fusionné.

==== Listes imbriquées `**` et `***`

*Fix* : Nouvelle méthode `isUnorderedListMarker(line)` reconnaît `* `,
`** `, `*** `. `listMarkerLength` retourne 2/3/4 selon le niveau.
`parseList` et `parseParagraph` utilisent cette méthode.

*Test* : `parses nested unordered list with double asterisk sub-items` —
4 items (parent + 2 sous-items + autre parent) dans une liste plate.

==== PlantUML pass-through

*Fix* : Branche `[plantuml]` dans `parseBlocks` redirige vers
`parseSourceBlock`. `parseSourceBlock` détecte `[plantuml]` et fixe
`language = "plantuml"`.

*Test* : `parses plantuml block as source-like pass-through` — vérifie
`language = "plantuml"` et contenu `@startuml...@enduml`.
