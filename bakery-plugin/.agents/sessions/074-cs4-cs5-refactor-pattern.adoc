== Décisions de conception

1. **ConfigInjector = fun interface** — pas de sealed class. Chaque injecteur est un `object` singleton sans état.
2. **resolve<T> avec 3 lambdas** — `parseCli`, `isExplicitlySet`, `isPresent`. Plus simple que sealed class ou fun interface pour les primitives.
3. **Scope pattern pour aggrégats** — `Scope` encapsule `props` + `prefix`, expose `string()`, `int()`, `boolean()`, `enum()`. Élimine la répétition `val prefix = "bakery.xxx"`.
4. **Theme et Firebase non touchés** — Clarity over cleverness. Leur logique spécifique (ThemeCatalog preset, `yaml?.project`) ne se prête pas à la généralisation simple.
