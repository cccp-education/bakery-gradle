== US-CR-V5-1 — Fix JsonEscape ordre remplacement (P0)

*Résultat* : FAUX POSITIF. L'ordre actuel `\`→`\\` avant `\n`→`\\n` est correct
pour du JSON produit par Jackson. Analyse caractère par caractère :

- Input Jackson : `\n` = 2 caractères (backslash+n, échappement JSON)
- `\`→`\\` transforme `\n` en `\\n` (3 chars) → properties → Jackson reparse `\n` = saut de ligne ✅
- Si on inversait, un saut de ligne littéral (0x0A) serait corrompu : `\n`→`\\n` insère un `\`,
  puis `\`→`\\` le double → `\\\n` ❌

Les 8 tests `JsonEscapeForPropertiesTest` passent avec les deux ordres (input Jackson uniquement).
Aucun changement code source.
