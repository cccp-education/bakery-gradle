== Livrables

=== 1. Bug parser `+` continuation — FIXÉ

*Problème* : `AsciiDocParser.parseParagraph` joignait les lignes avec un espace,
perdant les `+` de continuation AsciiDoc (hard line break). 15% du corpus
cheroliv.com impacté.

*Solution* :
- `PivotModel.kt` : ajout `PivotInline.LineBreak` (non-translatable)
- `AsciiDocParser.kt` : `buildParagraphInlines()` détecte `+` en fin de ligne,
  insère `LineBreak` entre les segments
- `JbakeNativeRenderer.kt` + `AsciiDocRenderer.kt` : `splitParagraphLines()`
  split sur `LineBreak` et rend ` +` en sortie
- `DocumentTranslator.kt`, `PivotYamlRenderer.kt`, `Content.kt` (×2) : branches
  `LineBreak` ajoutées

*Tests* : +5 tests (2 parser, 2 renderer JBake, 1 renderer AsciiDoc). 50/50 PASS
document-gradle, all PASS bakery-gradle.

=== 2. Gouvernance batch par année — DOCUMENTÉ

Stratégie batch par lot annuel documentée dans `BACKLOG.adoc` bakery (EPIC
BKY-I18N-BATCH). 8 lots (2019-2026), 108 articles, ~3h total avec parallelism=3.
