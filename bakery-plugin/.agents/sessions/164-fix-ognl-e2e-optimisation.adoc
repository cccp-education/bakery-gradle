== Taches realisees

=== 1. Fix bug OGNL #sets.contains dans 10 templates

**Probleme** : `#sets.contains({'ar','ur'}, ...)` echoue car `{...}` en OGNL cree
une `ArrayList`, pas une `Set`. `IllegalArgumentException: Unable to convert type
java.util.ArrayList of [ar, ur] to type of java.util.Set`.

**Fix** : `#sets.contains` -> `#lists.contains` dans 10 templates :
- site/templates/ : blog, index, page, post, archive, contact, hero_section, tags
- site-basic/templates/ : index, page

**Resultat** : 19/19 E2E PASS (avant : 17/19, 2 FAILED i18n)

=== 2. Cache installPlaywright (skip si Chromium present)

**Probleme** : La tache `installPlaywright` (NpxTask) se reexecutait a chaque
`./gradlew e2eTest` meme si les binaires etaient deja dans `~/.cache/ms-playwright/`.

**Fix** : Ajout `onlyIf` dans `build.gradle.kts` qui verifie la presence de
`chromium-*` dans le cache persistant. Zero marker file, zero dossier build.

**Resultat** : `installPlaywright SKIPPED` sur second run.

=== 3. Singleton PlaywrightPool (mutualisation instances lourdes)

**Probleme** : 5 classes E2E creaient chacune `Playwright.create()` + `chromium().launch()`.
Avec `forkEvery=1`, 5 JVMs separees => 5 startups Playwright + 5 launches Chromium.

**Fix** : `PlaywrightPool` singleton JVM-wide avec `@Synchronized` + shutdown hook.
`E2ETestBase` utilise `PlaywrightPool.getOrCreate()` au lieu d'instances locales.
`@AfterAll tearDownAll` supprime (cleanup via JVM shutdown hook).

**Resultat** : 1 seul `Playwright.create()` + 1 seul `chromium().launch()` pour tout.

=== 4. Suppression forkEvery=1 (1 JVM pour tout)

**Probleme** : `forkEvery = 1` dans `build.gradle.kts` forcait une JVM par classe.

**Fix** : Supprime. Toutes les classes E2E tournent dans 1 JVM, partageant le pool.

=== 5. Singleton ThymeleafRenderingTestFactory

**Probleme** : `ThymeleafRenderingTestFactory()` instancie a chaque appel `serveHtml()`
(19 fois).

**Fix** : Singleton dans le `companion object` de `E2ETestBase`.
