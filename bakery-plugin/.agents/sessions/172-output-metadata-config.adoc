== Décisions

* Héritage en cascade : enfant reçoit les champs non définis du parent
  (merge champ-par-champ, ancêtre le plus proche gagne)
* `NodeMetadataResolver` + `OutputConfigResolver` suivent le même pattern
  que `ThemeResolver.ancestorChain()`
* Champs optionnels (`null` = non défini, pas de valeur par défaut)
* Backward compat totale : `metadata: null` et `outputConfig: null` en YAML
  = comportement existant inchangé
* Règle de nommage : `OutputConfig` = configuration de *sortie* (ce qui est
  produit), `NodeMetadata` = métadonnées éditoriales (title, desc, tags)
