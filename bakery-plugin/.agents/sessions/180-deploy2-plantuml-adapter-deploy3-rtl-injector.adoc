=== DEPLOY-3 — `RtlDirectionInjector` (5 pts, DDD pur + UT + BDD)

*Fichier production* :
`src/main/kotlin/bakery/i18n/rtl/RtlDirectionInjector.kt` (37 lignes).

- `data class RtlDirective(lang: String, rtl: Boolean)` — value object.
- `class RtlDirectionInjector` — méthode `inject(frontmatter, lang)` :
  - Délègue à `LanguageCatalog.findByCode(lang)` (i18n-contracts MEM-3).
  - Lang inconnue → no-op (retourne frontmatter inchangé).
  - RTL (ar, ur) → `asciidocAttributes["jbake-lang"] = lang` +
    `["lang"] = "rtl"`.
  - LTR → `["jbake-lang"] = lang` + supprime `["lang"]` (idempotence
    inverse : source RTL traduite vers LTR drop RTL layout).
  - `:jbake-lang:` écrase la valeur précédente (idempotent).
- Inspiré du pattern `DeckTranslator.ensureRtlDirection` (MIAMI S030),
adapté au contexte bakery (attribut `:jbake-lang:` + `:lang: rtl` au lieu
de `:revealjs_direction: rtl`).

*UT* : `src/test/kotlin/bakery/i18n/rtl/RtlDirectionInjectorTest.kt`
(122 lignes, 11 UT, 4 `@Nested` groups : RtlInjection, UnknownLanguage,

=== DEPLOY-2 — `PlantUmlTranslationAdapter` (5 pts, DDD + adapter + UT + BDD)

*Fichier production* :
`src/main/kotlin/bakery/i18n/plantuml/PlantUmlTranslationAdapter.kt`
(93 lignes).

- `class PlantUmlTranslationAdapter(translationService, classifier)` —
  adapter qui dispatch sur `PlantUmlClassifier.classify` :
  - `PivotBlock.Source` non-plantuml → returned unchanged (`isSameAs`).
  - `PreserveTechnical` → block inchangé.
  - `TranslateLabels` → traduit chaque label (entre guillemets) via le port
    `TranslationService`. Les identifiants dotted (code qualifié) sont
    skippés upstream par `PlantUmlBlock.labels()`.
  - `BorrowVocabulary` → masque le vocabulaire métier (REAC, AFNOR, DC, TS,
    RNCP, CP, ECF) derrière des placeholders Unicode PUA (`\uE000...\uE001`)
    avant traduction, puis restaure verbatim. Le LLM ne peut pas réécrire
    le terme.
- I/O isolée via le port `TranslationService` (cf. `LlmServiceTranslationAdapter`
  S134). Domaine (`PlantUmlBlock`, `PlantUmlStrategy`, `PlantUmlClassifier`)
  reste pur. Inspiré HTOWN EPIC PLT-BOUNDARY (S163-166).

*UT* : `src/test/kotlin/bakery/i18n/plantuml/PlantUmlTranslationAdapterTest.kt`
(11 UT). Fake translators : `labelAware` (mapping FR→EN fixe) et `failing`
(`TranslationResult.Failure`). Tous GREEN.
