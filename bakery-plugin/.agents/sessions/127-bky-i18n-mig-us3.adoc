== DDD — Domain-Driven Design

`I18nMigrationService` — service métier pur, zéro dépendance Gradle :

- `scanTemplates(templatesDir)` → `List<File>` — walkTopDown, filtre `.thyme`
- `extractHardcodedText(templateFile)` → `MutableMap<String, String>` — 6 extracteurs :
  textContent, placeholder, ariaLabel, metaContent, alt, title
- `generateMessageFiles(allExtractions, languages, templatesDir)` → `Map<String, File>`
- `replaceHardcodedWithMessageKeys(templateFile, extractions)` → `File` — 6 apply* fonctions
- `writeMessageFiles(messageFiles, allExtractions)` — écrit Properties via `store()`
- `injectSiteLanguage(siteDir, defaultLanguage)` → `Boolean`
- `migrate(siteDir, languages, defaultLanguage, dryRun)` → `I18nMigrationResult`

`I18nMigrationResult` enrichi : +`templatesModified: Int`.

`MigrateToI18nTask` nettoyé : suppression `LlmService` (US-3 n'en a pas besoin,
US-4 viendra de codebase-gradle N1).
