== Réalisation

=== DDD — BfsProximityCache
Création d'un objet `BfsProximityCache` qui encapsule l'adjacency map et les distances BFS :
- `adjacency` : construit UNE FOIS à l'initialisation (plus de reconstruction par nœud)
- `proximityCache` : `MutableMap<String, Double>` avec lazy evaluation via `getOrPut`
- `proximityFor(nodeId)` : O(1) après premier calcul, même formule de proximité (dist0→1.0, dist1→0.7, dist2→0.4, dist3+→0.1)

=== TDD — 9 tests unitaires
`BfsProximityCacheTest` — 4 Construction + 4 Proximité BFS + 2 Performance :
- `cache contains all node ids` — tous les nœuds du sous-graphe
- `isolated node has proximity 1_0` — nœud seul
- `non-existent node has proximity 0_0` — nœud absent
- `empty subgraph has empty nodeIds` — sous-graphe vide
- `well connected node has proximity above zero` — nœud connecté
- `proximity is clamped between 0 and 1` — clamp [0,1]
- `central node has higher proximity than peripheral` — centralité
- `proximity matches existing computeGraphProximity` — **cohérence avec l'ancien calcul**
- `adjacency is built only once (idempotent)` — idempotence
- `all nodes accessible immediately` — accès immédiat

=== Refactor — AugmentedArticlesService
- `scoreAll()` construit `BfsProximityCache(subgraph)` UNE SEULE FOIS, puis appelle `scoreWithCache()` pour chaque nœud
- `score()` délègue à `scoreWithCache()` avec un cache créé à la volée (compat binaire préservée)
- Suppression de 3 méthodes privées : `computeGraphProximity()`, `bfsDistances()`, `buildAdjacency()` — déplacées vers `BfsProximityCache`
- `AugmentedArticlesService` passe de 321→239 lignes (-25%)

=== Complexité
- Avant : O(n²) — `buildAdjacency()` + BFS complet pour chaque nœud
- Après : O(n) — adjacency construit 1 fois, distances BFS lazy-cachées
