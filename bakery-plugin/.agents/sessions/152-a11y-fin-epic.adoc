=== Fix registrar (US-A11Y-4/5)

*Problème* : `AccessibilityTaskRegistrar` configurait `auditDir` via `layout.buildDirectory.dir("build/bake")` = `build/build/bake` (double `build/`), et `dependsOn(bake)` forçait l'exécution de JBake qui échouait sans `site/templates/` en TestKit. Aussi `reportPath` dérivait de `destDirPath` = `build/reports/build/bake-accessibility-audit.json` (inattendu).

*Corrections* :
. `auditDir` set via `layout.projectDirectory.dir(configuredDir)` (projectDir-relative, pas buildDir).
. `reportPath` default simple = `build/reports/accessibility-audit.json` (plus de dérivation destDirPath).
. `dependsOn(bake)` → `mustRunAfter(bake)` (l'audit ne force plus bake, s'exécute isolément).
. Cast `DirectoryProperty` inutile supprimé (warning Kotlin).

*Résultat* : Feature 37 scénarios compliant/non-compliant/empty → 4/4 PASS.
