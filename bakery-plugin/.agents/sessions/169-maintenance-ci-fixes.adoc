== Bugs résolus

=== Bug 1 — CI `./gradlew build` fail à settings evaluation (CRITIQUE)

*Fichier* : `settings.gradle.kts:23`
*Cause* : `error("ossrhUsername not found")` exécuté à TOUTE invocation Gradle
(build, test, tasks), pas seulement publish. CI n'a pas les credentials.
*Symptôme* : `BUILD FAILED in 5s` sur GitHub Actions — `ossrhUsername not found`
*Fix* : `?: ""` au lieu de `?: error(...)`. Credentials vides autorisés. L'échec
se produira uniquement si `publishAggregationToCentralPortal` est invoqué sans
credentials réelles.

=== Bug 2 — cucumberTest timeout 5min dépassé (CRITIQUE)

*Fichier* : `CucumberTestRunner.kt`
*Cause* : Feature 7 (CI/CD cheroliv.com) taguée `@personal` mais le runner ne
filtrait que `not @pipeline-contracts`. Les 3 scénarios `@personal` bakeaient la
vraie source cheroliv.com (655 fichiers, 12MB) via GradleRunner à chaque run,
dépassant le timeout 5min.
*Symptôme* : `./gradlew clean build` — `cucumberTest FAILED` après 8m22s.
*Fix* : `not @pipeline-contracts and not @personal` dans le filtre Cucumber.
Résultat : 193 tests, 14 skipped (was 11 — +3 @personal), 0 failures, 3m 4s.
Les scénarios `@personal` restent exécutables manuellement via
`-Dcucumber.filter.tags="@personal"`.

=== Bug 3 — validatePlugins fail Gradle 9.5 (BLOCKER)

*Fichier* : `AccessibilityAuditTask.kt`
*Cause* : `auditDir` annoté `@InputDirectory` sans `@PathSensitive`. Gradle 9.5
rejette les inputs non normalisés.
*Symptôme* : `./gradlew clean build` — `validatePlugins FAILED`.
*Fix* : `@get:PathSensitive(PathSensitivity.RELATIVE)` ajouté sur `auditDir` +
import `PathSensitive`/`PathSensitivity`.

=== Bug 4 — website.yml `org.jetbrains:annotations:{strictly 13.0}` (NON RÉSOLU)

*Cause* : Le POM publié de `codebase-plugin:0.0.2` porte une contrainte
`strictly 13.0` (Kotlin embedded) qui conflict avec `kotlinx-coroutines` (23.0.0)
et `flexmark` (24.0.1).
*Statut* : Hors périmètre bakery — nécessite republication de codebase-plugin
sans la contrainte `strictly`.
