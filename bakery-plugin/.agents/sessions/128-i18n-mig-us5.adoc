== Travail Réalisé

=== Feature 32_i18n_migration.feature — Enrichissement

7 scénarios avec assertions post-exécution :

. *@task-registered* — La tâche migrateToI18n est enregistrée (inchangé)
. *@dry-run-preview* — Dry run : vérifie qu'aucun fichier messages n'est créé,
  que les templates ne sont PAS modifiés, et que le log contient "DRY-RUN"
. *@migration-real-write* — Migration réelle EN+AR : vérifie que
  messages_en.properties et messages_ar.properties existent, que les
  templates contiennent th:text=, et que site.language=fr est injecté
  dans jbake.properties
. *@migration-no-templates* — Site sans templates : vérifie que le log
  contient "Aucun répertoire templates"
. *@migration-already-i18n* — Site déjà i18n : vérifie "Clés extraites : 0"
  et qu'aucun nouveau fichier messages n'est créé
. *@migration-no-hardcoded* — Site sans texte hardcodé : vérifie
  "Clés extraites : 0"
. *@migration-cli-options* — Options CLI : vérifie que --i18nSite,
  --i18nLangs, --i18nDryRun sont acceptés et que le log contient
  "Clés extraites" et "DRY-RUN"

=== Feature 31_i18n_migration.feature — Correction

Les 3 scénarios US-1 corrigés : pattern `dryRun true`/`false` littéral
→ `dryRun {word}` pour éviter l'ambiguïté Cucumber.

=== I18nMigrationSteps.kt — Nouveaux Steps

10 nouveaux steps (Given + And + Then/And) :

*Given* :
- `a real site directory {string} with templates containing hardcoded French text`
- `a real site directory {string} with NO templates directory`
- `a real site directory {string} with templates already using th:text message keys`
- `a real site directory {string} with templates containing only empty divs`

*And* :
- `i18n migration DSL configured with siteDir pointing to the real site and languages {string} and defaultLanguage {string} and dryRun {word}`

*When* :
- `I am executing the task {string} with CLI options for the real site and languages {string} and dryRun {string}`

*Then/And* :
- `no messages_{word}.properties file should exist in the templates directory`
- `the templates should NOT be modified`
- `messages_{word}.properties should exist in the templates directory`
- `the templates should contain th:text message keys`
- `site.language={word} should be injected in jbake.properties`
- `no new messages files should be created`

=== BakeryWorld.kt — Extension

- Champ `realSiteDir: File?` ajouté
- Méthode `createRealSiteDirectory(name, withTemplates, alreadyI18n, emptyDivs)`

=== Correction Bug — configureI18nMigrationDslWithRealSite

La première version appelait `createGradleProjectWithI18nMigrationIntention`
qui créait un NOUVEAU projet temporaire, écrasant `projectDir` et perdant
`realSiteDir`. Corrigé : réécriture directe du `build.gradle.kts` existant.

=== Correction Ambiguïté Cucumber

Les steps `dryRun true` et `dryRun false` (littéraux) créaient une ambiguïté
avec `dryRun {word}`. Supprimés. Feature 31 utilise maintenant `dryRun true`
(non quoté) matché par `{word}`.
