# Fichiers de Pilotage du Projet

Ce document répertorie les fichiers de pilotage principaux du projet, qui ont été créés pour organiser et structurer le contexte de développement.

## Fichiers de Contexte Principaux

*   **[GEMINI.md](GEMINI.md)**
    *   Description : Le fichier de pilotage principal du projet. Il contient l'aperçu général, les objectifs de développement, les exigences de qualité, le plan d'implémentation Scrumban (avec les épopées et les User Stories en cours), et un suivi des décisions de développement. Il sert de point d'entrée pour comprendre l'état actuel et les prochaines étapes du projet.

*   **[GEMINI.adoc](full_gemini_context.adoc)**
    *   Description : La version AsciiDoc du fichier de pilotage principal. Ce fichier est le résultat d'une conversion manuelle de `GEMINI.md` vers le format AsciiDoc, visant une meilleure intégration avec la stack de développement et les outils de l'utilisateur.

*   **[Roadmap du Projet](roadmap.adoc)**
    *   Description : Ce document AsciiDoc trace la vision à long terme, les phases de développement et les décisions stratégiques pour la refonte du site personnel. Il a été converti depuis `roadmap.md`.

## Fichiers de Contexte Détaillés

*   **[detailed_user_stories.md](detailed_user_stories.md)**
    *   Description : Contient les descriptions détaillées (objectifs, contraintes, modus operandi, résolutions) de toutes les User Stories du "Product Backlog" du projet. Ce fichier est consulté pour obtenir des informations approfondies sur chaque tâche.

*   **[GEMINI_archive.md](GEMINI_archive.md)**
    *   Description : Archive les User Stories et les tâches qui ont été terminées, ainsi que le bilan des revues de code et les améliorations apportées. Il permet de conserver un historique des travaux achevés sans surcharger le fichier de pilotage principal.

*   **[knowledge_base.md](knowledge_base.md)**
    *   Description : Centralise les informations techniques et les décisions de développement importantes, telles que les schémas de base de données, les configurations CI/CD et les justifications techniques. Il sert de référence pour les aspects techniques du projet.

*   **[iterative_development_guide.md](iterative_development_guide.md)**
    *   Description : Décrit le workflow itératif du projet (création, détaillage, implémentation, archivage des User Stories). Il contient également une section de guidance pour optimiser les routines de l'assistant IA Gemini-CLI.

*   **[site_static_statistics.md](site_static_statistics_roadmap.md)**
    *   Description : Une feuille de route pour la mise en place d'un système de surveillance et d'analyse pour le site web statique. Il détaille la stack technologique (Google Analytics, PageSpeed, Sentry, etc.), les métriques à suivre, les phases d'implémentation, et les configurations nécessaires pour monitorer le trafic, la performance, la disponibilité, et le comportement des utilisateurs.

*   **[iteration_collaboration_documentation.md](iteration_collaboration_documentation.md)**
    *   Description : Documente le processus de collaboration et de documentation pour la gestion des User Stories, en décomposant le travail en étapes claires (préparation, archivage, nettoyage) pour assurer la traçabilité et la cohérence du projet.

*   **[rag_action_plan.adoc](rag_action_plan.adoc)**
    *   Description : Détaille le plan d'action pour la mise en place de l'architecture RAG (Retrieval-Augmented Generation), en décomposant l'épopée 13 en User Stories et en actions concrètes.
