## 🎯 Résumé du Projet

**bakery-gradle** est un plugin Gradle qui orchestrera :
- 🔨 **JBake** : rendu statique (AsciiDoc → HTML)
- 🌳 **JGit** : publication git automatisée
- 📄 **GitHub Pages** : déploiement direct
- 🤖 **LLM (Ollama)** : génération de contenu assistée par IA
- 🔎 **Pagefind** : indexation full-text via npm

**Langue** : Kotlin (Gradle DSL)  
**Test Coverage** : 77.25% (seuil 75% ✅ dépassé)  
**Architecture** : Plugin pattern + TaskRegistrar delegation + Service layer

## 🏗️ Architecture Générale

```
BakeryPlugin (entry point)
    ↓
    ├→ BakeryExtension (DSL config)
    ├→ SiteConfiguration (YAML model)
    └→ TaskRegistrars (4 delegates)
         ├→ SiteTaskRegistrar (bake, serve, pagefind, config collection)
         ├→ DeployTaskRegistrar (git push: pages/maquette/profile)
         ├→ ContentTaskRegistrar (IA: article generation, theme, scaffolding)
         └→ LensTaskRegistrar (augmented context collection)
    
    Services layer:
    ├→ FileSystemManager (IO, resource copying)
    ├→ GitService (JGit wrapper)
    ├→ ProfilePublisher (GitHub profile updates)
    └→ LensPipelineService (Knowledge graph → context enrichment)
```

### Points Forts 👍

1. **Configuration externalisée** : YAML file (`site.yml`) + DSL extension = flexibilité
2. **Delegation pattern** : 5 registrars spécialisés au lieu d'une mega-classe (EPIC 11)
3. **Sealed classes** : `FileOperationResult`, `GitOperationResult` pour gestion d'erreurs explicite
4. **TDD/BDD** : Baby steps, tests imbriqués (@Nested), Cucumber features complètes
5. **CI robuste** : Coverage threshold 75%, GPG signing, multi-platform builds
6. **Type-safe Gradle** : Kotlin DSL + `Property<String>` pour lazy evaluation

#### **Ligne 226-236** : Gestion fragile du fichier augmented-context.json

```kotlin
val augmentedContextFile = projectDir.resolve("build/bakery/augmented-context.json")
if (augmentedContextFile.exists()) {
    val contextData = augmentedContextFile.readText(UTF_8)
        .replace("\\", "\\\\")      // 🔴 FRAGILE
        .replace("\n", "\\n")       // 🔴 FRAGILE
        .replace("\t", "\\t")       // 🔴 FRAGILE
    updateProperty(lines, "augmentedContextData", contextData)
} else {
    logger.info("[BakeryPlugin] augmentedContextData: file not found...")
    updateProperty(lines, "augmentedContextData", "")
}
```

**Risques** :
1. **Échappement JSON maison** : Les remplacements regex ne gèrent pas :
   - Caractères déjà échappés (`\"`)
   - Séquences Unicode (`\u0041`)
   - Guillemets imbriqués
2. **Perte de données** : Si le JSON contient `\n` littéral, il sera corrompu
3. **Pas de validation** : Aucune vérification que le JSON échappé est valide

**Solution recommandée** :
```kotlin
// Utiliser Jackson pour l'échappement correct
val contextData = augmentedContextFile.readText(UTF_8)
val escapedJson = jacksonObjectMapper()
    .writeValueAsString(mapOf("context" to contextData))
    .removeSurrounding("""{"context":"""", "}")

// OU : utiliser une property File directe
updateProperty(lines, "augmentedContextFilePath", augmentedContextFile.absolutePath)
// Dans la template Thymeleaf : `th:inline="javascript"` lit le fichier
```
